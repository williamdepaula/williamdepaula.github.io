Criando sites estático utilizando Flask

Introdução

O objetivo deste tutorial é mostrar como é fácil criar sites dinâmicos utilizando o Python e a Biblioteca Flask e depois transformar esse site em um site estático para ser hospedado facilmente em uma hospedagem de sites estáticos como o GITHUB Pages.

Pré-requisitos

  • Python 3.6 ou maior
  • Uma IDE para programação ou editor de texto ascii, recomendo o Visual Studio Code

Setup Inicial

Crie um pasta para manter os arquivos do seu projeto.Dentro dele crie um virtualenv python e instale os pacote Flask, Frozen-Flask e Flask-FlatPages

Projeto Inícial Flask

Vamos criar um arquivo server.py que conterá nosso projeto flask.

import sys, os
    from flask import Flask
    
    app = Flask(__name__)
    
    @app.route("/")
    def index():
        return "Olá Mundo!"
    
    if __name__ == "__main__":
        app.run(port=8000)

O que fizemos basicamente foi:

  • Linha 2: fazemos a importação da biblioteca flask
  • Linha 4: Criamos uma variavel que conterá nosso projeto flask
  • Linha 6: Definimos uma rota inicial do nosso site o /
  • Linha 7: Criamos uma função que será chamada quando nosa rota for acessada.
  • Linha 8: Basicamente nossa função retorna um Olá Mundo.
  • Linha 10: Quando nosso progrma for o programa principal ele é executado.
  • Linha 11: Roda nosso app flask na porta 8000

Para executar nosso projeto, com o virtualenv ativado, execute o nosso programa python.

$ python server.py 
     * Serving Flask app "server" (lazy loading)
     * Environment: production
       WARNING: This is a development server. Do not use it in a production deployment.
       Use a production WSGI server instead.
     * Debug mode: off
     * Running on http://127.0.0.1:8000/ (Press CTRL+C to quit

Agora abra o navegador no endereço http://localhost:8000/

É possível adicionar varias rotas ao nosso projeto, basta seguir o modelo:

@app.route("/bem-vindo")
    def bem_vindo():
        return "Seja bem-vindo!"

Isso criara uma rota http://localhost:8000/bem-vindo

Flask FlatPages

Flask-FlatPages é uma biblioteca que permite que sejam servidos pagina para uma aplicação Flask e que estas paginas seja geradas a partir de arquivos de texto plano("flat") em vez de banco de dados relacionais.
A ideia é que o flask entregue paginas html, utilizando como base arquivos markdown.

$ mkdir pages

Começamos criando uma pasta, chamada pages que irá armazenar todos as paginas em formato markdown que queremos disponibilizar.

Vamos criar um arquivo markdown de teste.
Crie um arquivo chamado pages/primeira-pagina.md como teste.

title: Primeira página de teste
    date: 2020-05-03
    
    # Titulo
    Este é o **primeiro** conteúdo da minha página!
    
    ## Sejá bem-vindo!

As linhas 1 e 2 são metadados que não será exibidos na página. Metadados são particularmente interessantes pois podem ser usados para organizar, filtrar, buscar seus textos. Por exemplo, se você quiser organizar suas postagens em ordem cronológica é necessário criar um metadado com a data da postagem, o mesmo se quiser organizar por TAGs e coisas do tipo.

Vamos alterar o nosso programa para servir está pagina:

import sys, os
    from flask import Flask
    from flask_flatpages import FlatPages
    
    DEBUG=True
    FLATPAGES_AUTO_RELOAD = DEBUG
    FLATPAGES_EXTENSION = ".md"
    
    app = Flask(__name__)
    app.config.from_object(__name__)
    pages = FlatPages(app)
    
    @app.route("/")
    def index():
        return "Olá Mundo!"
    
    @app.route("/<path:path>/")
    def page(path):
        return pages.get_or_404(path).html
    
    if __name__ == "__main__":
        app.run(port=8000)
  • Linha 3: Basicamente estamos importando o FlatPages
  • Linha 5: Criamos uma variável para dizermos que estamos em modo DEBUG.
  • Linha 6: Dizemos ao FlatPages que queremos que ele recarregue a pagina a cada nova requisição.
  • Linha 7: Todos os arquivos do tipo Markdown .md serão convertidos pelo FlatPages.
  • Linha 17: Criamos uma nova rota que qualquer nome que não caia na rota base, irá chamar nossa função page e passará como parâmetro o nome da rota.
  • Linha 18: Nossa função que recebe o nome da rota digitada, ex: se for localhost:8000/teste o parâmetro path receberá o valor "teste".
  • Linha 19: FlatPages procura um arquivo com o nome path.md seguindo o exemplo anterior, ele buscará o arquivo "teste.md" e o devolverá como html, ou um erro 404 se este arquivo não existir.

Execute novamente nosso programa e abra http://localhost:8000/primeira-pagina/

Template

Nosso programa está renderizando nossos arquivos .md corretamente, mas são paginas sem estilização css, e também para nosso site ficar mais coesso, queremos que ele renderize junto um cabeçalho, rodapé, sistemas de menu de navegação, etc...
Para isso o flask permite a criação de templates renderizados pelo Jinja2

$ mkdir templates

Primeiro criamos uma pasta chamada templates, por padrão o Flask busca todos os templates dentro dela.
Crie um arquivo chamado templates/base.html

<!doctype html>
    <html>
    <head>
        <meta charset="utf-8">
        <meta name="author" content="Site Pessoal">
        <meta name="description" content="Site com Flask">
        <title>Site com Flask</title>
    </head>
    
    <body>
        <h1>
            <a href="{{ url_for("index") }}">Site com Flask</a>
        </h1>
    
        <div>
            {% block content %}
                <p>Conteúdo Padrão</p>
            {% endblock content %}
        </div>
    </body>
    </html>
